# ttf2rastr_gfx Arduino Library

`ttf2rastr_gfx` is a small Arduino helper layer for drawing bitmap fonts generated by `ttf2rastr` through `Adafruit_GFX`.

The library is independent from any specific generated font. Only the font file is generated; the library itself stays reusable.

In the current version glyph rendering uses `Adafruit_GFX::drawBitmap()`, while the library handles metrics, placement, and the text-update workflow used by the example.

## Layout

- `library.properties`
- `src/ttf2rastr_gfx.h`
- `src/ttf2rastr_gfx.cpp`
- `examples/ESP32_ILI9341_FrameDemo/`
- `examples/ESP32_ILI9341_CyrillicDemo/`

## Local Installation

The easiest option:

- copy `Arduino/ttf2rastr_gfx` into your Arduino `libraries/` folder

After that, sketches can include:

```cpp
#include <ttf2rastr_gfx.h>
```

## What You Still Need

You also need a generated font file produced by `ttf2rastr`.

The recommended format is:

- `generated_font.h`

Typical setup:

```cpp
#include "generated_font.h"
#include <ttf2rastr_gfx.h>

static const RasterFont DEMO_FONT = TTF2RASTR_MAKE_FONT(demo_font);
```

## Public API

`ttf2rastrFindGlyph(const RasterFont* font, uint32_t codepoint)`

- finds a glyph by codepoint
- returns `NULL` when the glyph is missing

`ttf2rastrGlyphAdvance(const RasterFont* font, uint32_t codepoint)`

- returns horizontal advance for one symbol
- uses a fallback width when the glyph is missing

`ttf2rastrMeasureText(const RasterFont* font, const char* text)`

- measures a UTF-8 string
- returns `RasterTextMetrics`

`ttf2rastrTextHeight(const RasterTextMetrics* metrics)`

- converts measured vertical bounds into full text height

`ttf2rastrPlaceTextInBox(const RasterFont* font, int16_t box_x, int16_t box_y, int16_t box_w, int16_t box_h, const char* text)`

- computes centered text placement inside a rectangle
- returns `RasterTextPlacement`

`ttf2rastrDrawGlyph(Adafruit_GFX& display, const RasterFont* font, int16_t x, int16_t baseline_y, uint32_t codepoint, uint16_t color, uint16_t missing_color)`

- draws a single glyph
- uses `drawBitmap()`
- draws a fallback rectangle if the glyph is missing

`ttf2rastrDrawText(Adafruit_GFX& display, const RasterFont* font, int16_t x, int16_t baseline_y, const char* text, uint16_t color, uint16_t missing_color)`

- draws a full UTF-8 string
- returns the drawn width

## UTF-8 and Unicode

The library accepts ordinary UTF-8 `const char*` strings.

That means a sketch can simply do:

```cpp
ttf2rastrDrawText(tft, &font, x, baseline, "АБВ", ILI9341_YELLOW, ILI9341_RED);
```

A Cyrillic letter occupies multiple bytes in UTF-8, but glyph lookup is not done byte-by-byte.
The library first decodes UTF-8 into a single Unicode codepoint and then looks up the glyph by `uint32_t`.

So the flow is:

- `"А"` in UTF-8 -> bytes `D0 90`
- decode -> `U+0410`
- glyph lookup -> `ttf2rastrFindGlyph(font, 0x0410)`

For normal usage you do not need to hardcode `0x0410` yourself.
Just:

- generate a font that contains the needed symbols
- write UTF-8 text directly in the sketch source
- pass that string into `ttf2rastrDrawText()` or `ttf2rastrDrawTextInBox()`

`ttf2rastrDrawTextInBox(Adafruit_GFX& display, const RasterFont* font, int16_t box_x, int16_t box_y, int16_t box_w, int16_t box_h, const char* text, uint16_t text_color, uint16_t bg_color, uint16_t missing_color)`

- clears a rectangle
- centers the text inside it
- draws the whole string with one call

## Why The Font Is A Header

The generated font file does not contain executable logic. It only contains:

- bitmap data
- glyph tables
- font metrics
- related declarations

So using a header is more natural than compiling a separate `.cpp` file just for data.
